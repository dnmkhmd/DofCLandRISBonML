# Полный аудит и разбор программы: Система аренды и лизинга автомобилей (ML-Powered)

## 1. Обзор архитектуры
Проект представляет собой полнофункциональное веб-приложение для аренды и лизинга автомобилей с интеграцией машинного обучения для предсказания цен.

- **Frontend**: Next.js (React) + Vanilla CSS. Обеспечивает современный и отзывчивый интерфейс.
- **Backend**: FastAPI (Python) + SQLAlchemy. Высокопроизводительный API с асинхронной поддержкой.
- **База данных**: SQLite (по умолчанию `sql_app.db`).
- **ML**: Scikit-Learn (Random Forest) для оценки стоимости автомобилей на основе характеристик.

---

## 2. Структура проекта (Где что написано)

### Backend (`/backend`)
- `main.py`: **Центральный файл.** Содержит все эндпоинты API (машины, отзывы, калькуляторы, ML-предсказания), настройку CORS и загрузку ML-моделей.
- `models.py`: Определение таблиц базы данных (`User`, `Car`, `Booking`, `Review`, `Contact`).
- `schemas.py`: Pydantic-схемы для валидации данных на входе и выходе API.
- `database.py`: Конфигурация подключения к SQLAlchemy и создание сессии.
- `train_model.py`: Скрипт для обучения ML-модели на основе данных из `car_data.csv`.
- `car_data.csv`: Датасет для обучения модели.
- `models/`: Директория с сохраненными артефактами модели (`.pkl`).

### Frontend (`/frontend`)
- `src/app/page.tsx`: Главная страница с Hero-секции и популярными авто.
- `src/app/rent/page.tsx`: Страница каталога с фильтрацией (в процессе анализа).
- `src/components/`:
    - `CarCard.tsx`: Компонент карточки автомобиля.
    - `RentalCalculator.tsx`: Модальное окно расчета стоимости аренды.
    - `LeasingCalculator.tsx`: Калькулятор ежемесячных платежей по лизингу.
    - `ui/Button.tsx`: Универсальный компонент кнопки.
- `src/utils/api.ts`: Настройка Axios и функции для взаимодействия с Backend.

## 3. Подробный аудит и выявленные ошибки

### ❌ Backend (Критические и средние)

1.  **Хрупкая загрузка ML-моделей (`main.py`):**
    - *Где:* Строки 36-39.
    - *Ошибка:* Если файлы модели отсутствуют, приложение просто выводит принт и продолжает работу. При попытке вызвать `/predict_price` вернется 503.
    - *Исправление:* Добавить логирование ошибок и, возможно, автоматический запуск `train_model.py`, если модели нет.

2.  **Проблема парсинга дат (`main.py`):**
    - *Где:* Строки 131-132 в `calculate_rent`.
    - *Ошибка:* Использование `.replace("Z", "+00:00")` рассчитано строго на ISO-строку с Z. Если фронтенд пришлет просто дату `YYYY-MM-DD`, код может упасть на `replace` или на `fromisoformat`.
    - *Исправление:* Использовать библиотеку `dateutil` или более гибкий парсинг.

3.  **Небезопасный CORS (`main.py`):**
    - *Где:* Строка 22 (`allow_origins=["*"]`).
    - *Ошибка:* Разрешены запросы с любых доменов.
    - *Рекомендация:* Ограничить доверенными доменами в продакшене.

4.  **Устаревший метод Pydantic (`main.py`):**
    - *Где:* Строка 88, 109, 119 (`car.dict()`).
    - *Ошибка:* В Pydantic v2 рекомендуется `model_dump()`.

5.  **Отсутствие миграций:**
    - *Ошибка:* База создается через `create_all`. При изменении моделей данные придется удалять или править вручную.
    - *Рекомендация:* Использовать `Alembic`.

### ⚠️ Frontend (Интерфейс и логика)

1.  **Хардкод URL API (`api.ts`, `RentalCalculator.tsx`):**
    - *Где:* `baseURL: 'http://localhost:8000'`.
    - *Ошибка:* При деплое приложение сломается.
    - *Исправление:* Вынести в `.env.local` переменную `NEXT_PUBLIC_API_URL`.

2.  **Некорректное отображение трансмиссии (`CarCard.tsx`):**
    - *Где:* Строка 44 (`car.transmission[0]`).
    - *Ошибка:* Если в базе строка "Automatic", отобразится просто буква "A". Это выглядит как баг.
    - *Исправление:* Выводить полную строку или использовать маппинг.

3.  **Эффективность запросов (`page.tsx`):**
    - *Где:* Запрос всех машин для отображения только первых трех.
    - *Исправление:* Добавить параметр `limit=3` в вызов API.

4.  **Дублирование логики API (`RentalCalculator.tsx`):**
    - *Где:* Прямой вызов `axios` вместо использования `api.ts`.
    - *Исправление:* Переиспользовать инстанс из `utils/api.ts`.

5.  **Несоответствие тестов и схем (`tests/test_main.py`):**
    - *Ошибка:* Тесты используют поля `make` и `price`, в то время как Pydantic-схемы и модели используют `brand` и `rent_price_day`.
    - *Следствие:* Существующие тесты, скорее всего, не проходят или тестируют устаревшую логику.
